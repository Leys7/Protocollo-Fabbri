### 🕵️‍♂️ Panoramica del Caso
Il progetto è una piattaforma didattica interattiva basata su un'indagine forense. Gli utenti vestono i panni di un analista **OSINT** (Open Source Intelligence) incaricato di indagare su uno studente, Luca Fabbri, accusato di aver violato i server della scuola.

L'esperienza si snoda attraverso diverse fasi di analisi:
1. **Phishing Detection:** Analisi delle intestazioni mail.
2. **OSINT Geografico:** Ricostruzione delle abitudini tramite social media.
3. **Log Analysis:** Analisi dei registri server per identificare discrepanze IP e User-Agent.
4. **Kill Switch:** Validazione finale dei dati raccolti.

---

## 🛠️ Tecnologie Utilizzate
Per garantire la massima compatibilità e performance (specialmente su piattaforme come GitHub Pages), il progetto è stato sviluppato senza librerie pesanti:

* **HTML5 & CSS3:** Layout responsive con estetica *Cyber-Dark*.
* **Vanilla JavaScript (ES6+):** Motori di calcolo per le collisioni (Hitbox), algoritmi di decriptazione e logiche di validazione.
* **Google Sheets API (Preview):** Integrazione dinamica dei log di sistema per simulare database reali.

---

## 📂 Architettura dei File
La navigazione è strutturata in nodi sequenziali protetti da password (le chiavi recuperate durante l'indagine):

* `index`: Punto di ingresso e analisi della stanza (Hitbox JS).
* `livello_003.`: Terminale di Bruteforce e analisi social.
* `livello_004.`: Dashboard forense e analisi dei log IP.
* `epilogo`: Protocollo Cassandra e chiusura del dossier.
